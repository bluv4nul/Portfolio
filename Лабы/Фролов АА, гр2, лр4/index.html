What is Open Source Software

Open Source Software (OSS) — software whose source code is available for viewing, modification, and distribution.

The concept supports collaboration, innovation, and community-driven development.

Principles of Free Software (by Free Software Education):

  1. Freedom to use the software for any purpose.
  2. Freedom to study the source code and adapt the software.
  3. Freedom to distribute copies.
  4. Freedom to improve the software and share the modifications

What are Software Licenses?

A software license is a legal agreement that defines how the software can be used, modified, and distributed.

Licenses help clarify the relationship between developers, users, and the software

Main categories of licenses:

  1. Free and Open Source Licenses (e.g., GPL, MIT, Apache) — These promote collaboration and redistribution.
  2. Proprietary Licenses — Users are allowed to use the software under strict conditions, often without modification rights.
  3. Creative Licenses — Designed for content rather than software, such as Creative Commons licenses.

For an in-depth guide to various licenses, visit Choose a lisence.

GPL 3 (General Public License, Version 3)

The GNU General Public License (GPL), developed by the Free Software Foundation, is one of the most popular open-source licenses.

Key features of GPLv3:

  1. Ensures that anyone can use, modify, and redistribute the software.
  2. Requires that any distributed modified versions also remain open-source.
  3. Prohibits incorporating the software into proprietary systems without releasing the source code.

Example: Software like WordPress and GIMP are licensed under GPL. For more, explore the official GPL page.

Creative Commons

Creative Commons (CC) licenses are commonly used for educational materials, media content, and other non-software resources.

There are six main types of CC licenses:

  1. CC BY: Attribution required; allows any use, even commercially.
  2. CC BY-SA: Attribution + ShareAlike; derivatives must use the same license.
  3. CC BY-ND: Attribution + No Derivatives; no changes allowed.
  4. CC BY-NC: Attribution + Non-Commercial; prohibits commercial use.
  5. CC BY-NC-SA: Attribution + Non-Commercial + ShareAlike.
  6. CC BY-NC-ND: Attribution + Non-Commercial + No Derivatives; most restrictive license.

For more details, visit the Creative Commons website.

Recommendation for Educational Materials

When creating educational materials, such as course projects or theses, it’s important to choose an appropriate license.

Recommended License: CC BY-NC

  1. Allows others to use and adapt your work freely.
  2. Ensures your authorship is acknowledged.
  3. Prohibits commercial use, protecting the material from unauthorized exploitation.

To learn how to apply a Creative Commons license to your work, check out this guide.

Conclusion

Understanding software licenses ensures compliance with legal requirements and helps foster innovation and collaboration.

Choose licenses based on the purpose of the work:

  1. GPL for software projects.
  2. Creative Commons for educational or multimedia resources.

Proper licensing protects your work and clarifies its intended use for others.
